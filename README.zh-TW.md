> **GridPlayer** 是一個 Chrome 擴充功能（Manifest V3），把瀏覽器的 **側邊欄（Side Panel）** 變成一面影片牆。貼上最多 **25** 個影片網址，就能在 **2×2 / 3×3 / 4×4 / 5×5** 格狀畫面中同時播放——不用一直切分頁、不用手動排視窗。它會自動把一般網頁網址轉成 embed、把你常看的組合記成一鍵還原的群組，每一格還都有獨立的播放控制。支援 YouTube、Twitch、Vimeo、Nicovideo、TikTok 等多個平台。

---

## 痛點

在網路上「同時看好幾個東西」其實很麻煩：

- **分頁地獄。** 一個串流開一個分頁，結果一直來回點——同一時間永遠只看得到一個。
- **子母畫面只能一個。** PiP 只能彈出*一個*影片，沒辦法同時排四場比賽、三個音樂直播加一支教學。
- **手動排版。** 自己拖拉、縮放一整排瀏覽器視窗很累，而且永遠對不齊。
- **嵌入封鎖。** 很多播放器拒絕在 `<iframe>` 載入，就算自己土法煉鋼排格子，一半的網站也會掛掉。

最需要「多視窗儀表板」的人——體育迷、投資人、電競觀眾、學生——最後都在跟瀏覽器搏鬥，而不是在看影片。

## 解法

**GridPlayer 把這一切收進一個側邊欄。**

```
貼上網址  →  選 2×2 / 3×3 / 4×4 / 5×5  →  全部同時播放
                          ↓
        把這組存成命名群組 · 之後一鍵還原
```

貼上一般的影片頁面網址，GridPlayer 會自動幫你算出 embed；它會移除原本會擋住播放的框架標頭，把影片排成乾淨的格狀畫面，再給你每一格獨立的控制——讓你終於可以在不離開當前頁面的情況下,同時看很多東西。

## 核心功能

- 📺 **最多 25 部影片同時播放** —— 2×2、3×3、4×4、5×5 格局，隨時切換。
- 🔗 **貼上即播** —— 一般網頁網址自動轉成 embed，不用自己找嵌入碼。
- 📌 **群組記憶** —— 把目前組合存成命名群組，一鍵還原（最多 20 組）。可加備註、重新命名、修改 URL，或快速儲存覆蓋。
- 🖱️ **到處都能拖** —— 拖曳重排、把外部連結直接拖進格子、一鍵重排消除空格。
- 🎛️ **每格個別控制** —— 每一格都有獨立的播放、暫停、重載、停止、下一部、在新分頁開啟按鈕。
- 🔊 **全域控制** —— 一鍵暫停全部 / 繼續全部、靜音全部 / 取消靜音。
- 📋 **複製與批次貼上** —— 一鍵複製所有網址,或一次貼入多個 URL 填滿空格。
- 🎯 **選取模式** —— 在受支援的網站上會出現浮動工具列,點選影片縮圖直接送入面板。
- 🪟 **在新分頁開啟** —— 把格狀畫面搬到獨立視窗,適合大螢幕（自動調整至最低 900px）。
- 🌐 **多語言介面** —— 繁中、简中、English、日本語,隨時切換。
- 🎨 **亮色 / 暗色主題** —— 設定會記憶。
- ⚡ **輕量無廣告** —— 純側邊欄設計,不注入廣告、不追蹤行為。

## 為什麼選 GridPlayer

|  | 多開分頁 | 子母畫面（PiP） | **GridPlayer** |
|---|---|---|---|
| **同時可見的影片數** | 只有當前分頁 | 1 個浮動視窗 | **格狀最多 25 個** |
| **格局** | 手動排視窗 | 無 | **2×2 / 3×3 / 4×4 / 5×5,一鍵** |
| **儲存與還原組合** | ❌ | ❌ | **✅ 命名群組,一鍵還原** |
| **畫面內個別控制** | 要先切分頁 | 有限 | **✅ 每一格都有** |
| **跨平台嵌入** | — | — | **✅ YouTube、Twitch、Vimeo… 自動轉換** |
| **佔用** | 重（N 個分頁） | 內建 | **一個側邊欄** |

**關鍵差異:** 只有 GridPlayer 能在「可控的格狀畫面」裡顯示多個影片,*又*能記住你的設定——而且完全不用多開任何一個分頁。

## 安裝方式

### 1. Chrome Web Store（建議）

> 🚧 上架審核中 —— 連結即將提供。

### 2. 開發者模式載入

1. 下載或 clone 本專案到本機任意資料夾。
2. 在 Chrome 網址列輸入 `chrome://extensions` 並進入。
3. 開啟右上角的 **「開發人員模式」** 開關。
4. 按下左上角的 **「載入未封裝項目」**。
5. 選取本專案的資料夾（包含 `manifest.json` 的那一層）。
6. 安裝完成後,Chrome 工具列會出現 GridPlayer 圖示。

## 如何使用

### 開啟側邊欄
點一下工具列的 **GridPlayer 圖示**,右側會滑出側邊欄。

### 貼上影片 URL
1. 在任意一格點 **「＋」** 按鈕,輸入框會展開。
2. 貼上影片頁面的網址（例如 `https://www.youtube.com/watch?v=xxxxx`）。
3. 按 **播放** 或 **Enter**,影片就會在該格載入。

支援直接貼上影片頁面的一般網址,不需要自己找 embed 連結——擴充功能會自動轉換。

### 批次貼上
把多個 URL 一行一行複製到剪貼簿,然後按頂部的 **「貼上」** 按鈕,系統會依序填滿空格。

### 格局切換
頂部有 **2×2 / 3×3 / 4×4 / 5×5** 四個按鈕,隨時切換格數。

### 選取模式（在影片網站上使用）
在受支援的影片網站瀏覽時,頁面右下角會出現一個浮動工具列：

- 按 **「選擇」** 進入選取模式。
- 點擊頁面上的影片縮圖連結,即可選取（會以紫色框標示）。
- 選好後按 **「送入面板 →」**,側邊欄會自動開啟並載入選取的影片。

## 技術內幕（Under the hood）

為了嵌入那些原本會抵抗的播放器,同時維持輕量的純側邊欄設計：

- **Manifest V3 + Side Panel API** —— 整個介面都在 Chrome 原生側邊欄,不注入頁面 UI、不佔背景分頁。
- **URL 自動轉換**（`url_converter.js`）—— 貼上一般影片頁面,自動依各平台算出正確的 embed URL。
- **以 `declarativeNetRequest` 移除框架標頭** —— 移除 `X-Frame-Options` / frame-ancestors 等標頭,讓會擋 `<iframe>` 的播放器能正常載入。規則刻意拆成數條較短的 regex,以符合 Chrome 對 `regexFilter` 的複雜度上限（單一過長 regex 會靜默地註冊失敗）。
- **Twitch 嵌入繞過** —— 透過 DNR 改寫讓 Twitch 播放器接受 `parent=localhost` 請求,使頻道、VOD、Clips 都能在格子裡播放。
- **Content Script + 選取模式** —— 在受支援網站提供浮動工具列挑選縮圖送入面板,並針對特定網站修正（例如隱藏 XHamster 蓋在播放鍵上的 CTA 覆蓋層、攔截 `gamr.info` 導向連結）。
- **執行期 i18n**（`i18n.js`）—— 不重新載入面板即可在 繁中 / 简中 / EN / 日本語 之間切換。

## 常見問題（FAQ）

**Q: 影片格子顯示錯誤頁面或空白,怎麼辦？**
先試試每格右側的「重載」按鈕。若仍無法播放,該平台可能暫時不支援嵌入。

**Q: YouTube 影片無法自動播放？**
Chrome 的自動播放政策要求使用者先與頁面互動。點一下影片格的播放鍵即可。

**Q: Twitch 直播播不出來？**
本擴充功能使用 `parent=localhost` 參數搭配 DNR 規則來符合 Twitch 的 embed 限制,大多數情況下可正常播放。若仍有問題,請確認 Twitch 頻道名稱拼寫正確。

**Q: 群組最多可以存幾個？**
最多 **20 組**。超過上限時會出現提示,請先刪除舊群組。

**Q: 切換語言後介面沒有完全更新？**
動態產生的群組名稱會在切換語言後自動重新渲染。若有其他元素未更新,重新整理側邊欄即可。

**Q:「在新分頁開啟」和直接使用側邊欄有什麼差別？**
功能完全相同,差別只在視窗形式。新分頁模式可以拉更大的視窗,適合大螢幕使用,視窗寬度會自動記憶（最低 900px）。

## 已知限制

- **XVideos / XNXX** —— 這兩個平台的 embed 頁面設有 `X-Frame-Options: sameorigin` 限制,目前無法完全繞過,可能顯示錯誤。URL 仍會被轉換為 embed 格式,但能否實際播放取決於平台當時的政策。
- **TikTok** —— 平台限制使部分影片無法嵌入。
- **自動播放** —— 需先與頁面互動才會播放（Chrome 政策),點該格播放鍵即可。
- **群組上限 20 組** —— 超過請先刪除舊群組。
